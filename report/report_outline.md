# NorthStar Databases and Analytics Assignment Report Outline

## 1. Introduction

This report presents the design and implementation of a database and analytics solution for NorthStar Urban Mobility and Logistics. The project demonstrates relational database design, SQL querying, query optimisation, Python data processing, R analytics, and MongoDB document-based querying.

## 2. Database Design

The relational database includes the following tables:

- Customers
- Drivers
- Vehicles
- Routes
- Deliveries

The design uses primary keys, foreign keys, and constraints to maintain data integrity.

## 3. SQL Implementation

The SQL implementation includes:

- Database creation
- Table creation
- Sample data insertion
- Analytical queries
- Join operations
- Aggregation queries
- Query optimisation using indexes

## 4. Python Data Processing

Python is used to load, clean, and summarise logistics data from the sample CSV dataset.

## 5. R Analytics

R is used to generate descriptive statistics, delivery status summaries, average distance calculations, and basic visual analysis.

## 6. MongoDB Implementation

MongoDB is used to demonstrate document-based storage and querying for delivery records.

## 7. Query Optimisation

Indexes are created on frequently queried fields such as delivery status, delivery date, customer ID, driver ID, and vehicle ID.

## 8. Conclusion

The project demonstrates how database and analytics tools can support operational decision-making in an urban logistics environment.
